# Icon Usage Policy

**Status**: Authoritative (single source of truth — do not duplicate elsewhere)
**Owner**: Growth FE (STOA-Foundation) **Origin**: PG-195 /
[ADR-046](../architecture/adr/ADR-046-material-symbols-font-subsetting.md)
**Last updated**: 2026-04-15

This document is the **only** place where the icon policy is defined. Every
other doc, CLAUDE.md, skill, or review checklist that mentions icons MUST link
here instead of restating rules.

---

## The rule

Icons in IntelliFlow CRM render through **Material Symbols Outlined** and
nothing else. The three allowed shapes are:

```tsx
// 1. Direct ligature (canonical)
<span className="material-symbols-outlined" aria-hidden="true">arrow_back</span>

// 2. Shared wrapper (preferred for uniform sizing + a11y)
<Icon name="arrow_back" />
<IconBadge icon="check_circle" tone="success" />

// 3. Via the canonical mapping
import { ICON_MAPPING } from '@intelliflow/ui/lib/icon-mapping';
<span className="material-symbols-outlined">{ICON_MAPPING.ChevronDown}</span>
```

Icon names are the lowercase-snake ligatures Google publishes — see
<https://fonts.google.com/icons?icon.set=Material+Symbols&icon.style=Outlined>.

## What is forbidden

- `lucide-react` — **hard error** via ESLint `no-restricted-imports` (package
  removed from `apps/web/package.json` on 2026-04-15 after the migration
  finished).
- `@heroicons/react` — `error`.
- `react-icons` — `error`.
- `@radix-ui/react-icons` — `error`.
- `react-feather` — `error`.
- Inline SVG `<svg>` "icons" (brand illustrations, logos, and decorative
  graphics are fine — but anything that's semantically an icon must use Material
  Symbols).

The ban is enforced in `apps/web/eslint.config.mjs`. Adding a new foreign icon
package fails CI at the `Lint` gate before the material-symbols audit even runs.

## Pinned variation axes

Production CSS pins a single static point
(`apps/web/src/app/globals.css:239-260`):

| Axis   | Value |
| ------ | ----- |
| `wght` | `400` |
| `FILL` | `0`   |
| `GRAD` | `0`   |
| `opsz` | `24`  |

The shipped woff2 is a **static instance** at those values — the variable axes
have been dropped to minimize byte weight. Do NOT add CSS that overrides
`font-variation-settings` or introduce multiple weights of the font; if you need
a different look, open an ADR superseding ADR-046.

## The available icon set

- **Authoritative list**: `artifacts/reports/material-symbols-glyph-audit.json`
  field `icons[]` (359 glyphs as of 2026-04-15).
- **Generated by**: `tools/scripts/subset-material-symbols.mjs` (see
  `tools/scripts/README.md`).
- **Source of truth for future additions**: this file — every icon named in the
  audit must appear as either a ligature literal in source, a value in
  `ICON_MAPPING`, or an `icon: 'name'` / `icon="name"` property harvested by the
  scanner.

## How to add a new icon

Two paths:

### Path A — icon is already in the upstream Material Symbols font (usual case)

1. Use the ligature at a call site:
   `<span className="material-symbols-outlined">my_new_icon</span>`.
2. Run the subsetter locally:
   ```bash
   node tools/scripts/subset-material-symbols.mjs
   ```
3. Commit the regenerated `apps/web/public/fonts/MaterialSymbolsOutlined.woff2`
   and `artifacts/reports/material-symbols-glyph-audit.json` together.
4. CI's `Audit Material Symbols glyph coverage` step (pr-checks.yml + ci.yml)
   will now pass. If you skip step 2/3, CI fails fast with a diff of missing
   icons.

### Path B — icon does NOT exist in upstream Material Symbols (rare)

Material Symbols ships ~3,500 glyphs — step B is rare. If you need an icon that
genuinely doesn't exist:

1. Follow the
   [ADR-046 Rollback Plan](../architecture/adr/ADR-046-material-symbols-font-subsetting.md#rollback-plan)
   to bump the committed upstream fixture at
   `tools/scripts/fixtures/MaterialSymbolsOutlined-upstream.woff2` to a newer
   Google release.
2. Re-run the subsetter.
3. If even the latest upstream lacks the icon, open a design discussion before
   inventing a custom glyph — a bespoke SVG illustration is usually the right
   answer for one-off branded imagery, NOT a new icon.

## Outstanding migrations

**None.** The lucide-react migration finished on 2026-04-15. Production code in
`apps/web/src/**` and `packages/ui/src/**` imports zero foreign icon libraries;
the `lucide-react` package has been removed from `apps/web/package.json`; the
ESLint transitional `warn`-override block was removed from
`apps/web/eslint.config.mjs`. Net-new imports of any banned library now fail at
lint time with no exceptions.

## Why Material Symbols only

- **Lighthouse Performance ≥90**: the full Material Symbols variable font is
  3.47 MB and drives Lighthouse Performance on `/404` and `/500` down to 0.63
  (PG-056 baseline). The subsetted static instance is 234 KB.
- **Single-axis pin**: dropping `FILL`, `GRAD`, `opsz`, and the weight range
  yields ~93% byte savings.
- **Zero visual regression**: every call site renders from one consistent font;
  no mix of stroke widths or metrics from competing libraries.
- **Single CI guard**: `--verify` ensures every rendered icon is in the audit,
  preventing tofu regressions. A mixed-library world would need a separate guard
  per library — none exist for SVG component packages.

## How the guard works (one-paragraph summary for reviewers)

`tools/scripts/subset-material-symbols.mjs --verify` scans every
`.ts/.tsx/ .js/.jsx` file under `apps/web/src/**` and `packages/ui/src/**`
(excluding tests, stories, and docs), harvests icon names from five shapes (JSX
ligature children, `icon:` property literals, `iconName=` JSX props,
`cn(...)`/`clsx(...)`-wrapped className spans, template-literal className),
unions them with every `ICON_MAPPING` value, and diffs against
`artifacts/reports/material-symbols-glyph-audit.json`. Any source-discovered
icon that isn't in the audit exits 1 with a diff. The guard runs on every PR in
`.github/workflows/pr-checks.yml` (Quality Checks job) and on every main-branch
build in `.github/workflows/ci.yml` (before the `Build` step).
