# CTA Library — IntelliFlow CRM

**Purpose**: Approved call-to-action copy for use in the pitch deck, landing
pages, email sequences, and in-product prompts. Always pull from this list
before writing new CTAs to maintain voice consistency.

---

## Primary CTAs (high intent — use for main conversion actions)

| CTA Text           | Context                       | Destination                   |
| ------------------ | ----------------------------- | ----------------------------- |
| Request a demo     | Pitch deck, landing page hero | `/demo` form or Calendly link |
| Join the pilot     | Pitch deck slide 12, emails   | `/pilot` application form     |
| Get early access   | Blog posts, outbound emails   | `/early-access` waitlist      |
| Start for free     | Pricing page (future)         | `/signup`                     |
| Book a 30-min call | Sales emails, LinkedIn        | Calendly scheduling link      |

## Secondary CTAs (lower friction — use for nurturing actions)

| CTA Text                     | Context                         | Destination                        |
| ---------------------------- | ------------------------------- | ---------------------------------- |
| See the roadmap              | Existing leads, pilot follow-up | Public roadmap URL (TBD)           |
| View documentation           | Post-demo follow-up, engineers  | `https://docs.intelliflow-crm.com` |
| Read the case study          | Late-stage nurture (future)     | `/case-studies/` (not yet live)    |
| Explore the architecture     | Engineering leads               | `docs/architecture/` or docs site  |
| See how the validation works | RevOps / Ops leads              | Pitch deck slide 5 or demo         |

## Persona-Specific Variants

### Founder / GM

- "See predictable execution in action" → demo
- "Join the pilot — free for 3 months" → pilot form

### Sales Lead

- "Cut CRM admin by 50%" → demo
- "See automated follow-up in 5 minutes" → demo

### RevOps / Ops

- "Get dashboards you can trust" → demo
- "See validation gates live" → demo

### Engineering Lead

- "Explore the open architecture" → architecture docs
- "See the type-safe API in 10 minutes" → demo

---

## Tone & Style Rules

- Use active voice and present tense.
- Avoid weak verbs: "learn more", "click here", "find out".
- Prefer concrete outcomes: "See…", "Get…", "Join…", "Build…".
- One CTA per surface. Never stack two primary CTAs on the same page/slide.
