# Feature Matrix (Source of Truth)

Last audited: 2026-02-08  
Owner: Product + Engineering

## Purpose

Single artifact to track feature implementation state and delivery completeness across IntelliFlow CRM.

This matrix is the canonical view for:
- Feature status (`planned`, `in_progress`, `done`)
- Requirement traceability (`Related Task (CSV)`, `Flow Ref`, `PRD Ref`, `ADR Ref`)
- End-to-end chain coverage (`Entity -> Domain -> Database -> Adapter -> Router -> Frontend`)
- Validation sources (`KPI`, `Validation Method`, evidence artifacts)
- Missing CSV tasks needed to fully plan or complete a feature

## Status Rubric

| Status | Definition |
|---|---|
| `done` | Implemented and wired to active data/API paths for the primary user flow |
| `in_progress` | Partially implemented, mock/placeholder behavior, or known integration gaps |
| `planned` | Backlog item with no production-ready flow yet |

## Completeness Rubric

| Plan Coverage | Definition |
|---|---|
| `complete` | All required layers for this feature type are covered by CSV tasks |
| `partial` | 1-2 required layers are missing tasks |
| `gap` | 3+ required layers are missing tasks |
| `at_risk` | Feature is `done` or `in_progress` but still has missing required layers |

Layer cell legend:
- `n/a` = layer is not required for this feature type
- `missing` = required layer has no linked CSV task
- `done/in_progress/planned/partial (...)` = layer has linked task coverage
